# v4.1 Prompt Changes — All Verticals

**Date**: 2026-03-12
**Dataset**: 1,227 leads (734 SEW + 493 Venture), 691 matched to dispo, 617 contacted, 88 appointments, 3 sales

## Changes Applied (both solar AND roofing prompts)

### 1. Grade F + Low Activity → Bronze Cap (ALL VERTICALS)
- **Signal**: phone.contact_grade = "F" AND activity_score < 40
- **Data**: 0/15 appointments (0%), 60%+ DQ rate across combined dataset
- **Change**: Cap at Bronze regardless of other signals. If also NonFixedVOIP → Reject.
- **Impact**: 24 leads had this combo in backtest

### 2. NonFixedVOIP → Bronze Cap (ALL VERTICALS)
- **Signal**: phone.line_type = "NonFixedVOIP"
- **Data**: 0/12 appointments (0%), 50% DQ rate
- **Change**: Cap at Bronze. Combined with Grade F or low activity → Reject.
- **Impact**: 19 leads had NonFixedVOIP in backtest

### 3. FixedVOIP → Moderate Concern (ALL VERTICALS)
- **Signal**: phone.line_type = "FixedVOIP"
- **Data**: 1/12 appointments (8.3%), 58.3% DQ rate
- **Change**: Upgraded from "slight concern" to "moderate concern". Excluded from Gold.
- **Impact**: 17 leads had FixedVOIP

### 4. Grade F + Activity >= 40 → Bronze/Low Silver Only (ALL VERTICALS)
- **Signal**: phone.contact_grade = "F" AND activity_score >= 40
- **Data**: Small sample but no appointments observed
- **Change**: Score Bronze or low Silver ONLY if identity + property very strong
- **Impact**: ~1-2 leads

## Changes Applied (solar prompt ONLY)

### 5. Solar Permit → Bronze Cap (SOLAR ONLY)
- **Signal**: batchdata.solar_permit = true
- **Data**: 0/16 appointments (0%), 62.5% DQ rate
- **Change**: Explicit Bronze cap. Was "strong downgrade" but LLM scored 11/16 as Silver.
- **Roofing equivalent**: roof_permit already has Bronze cap in roofing prompt

### 6. Commercial Property → Neutral (SOLAR ONLY)
- **Signal**: batchdata.property_type = "Commercial"
- **Data**: 14 contacted, 3 appointments (21.4% — ABOVE 14.3% base rate). LLM was rejecting these as "can't install residential solar" but they actually outperform.
- **Change**: Changed from implicit reject/concern to explicit NEUTRAL. "Do NOT penalize or reject commercial property leads."
- **Roofing**: Kept as "strong negative" — residential roofers genuinely can't serve commercial buildings
- **Impact**: Recovers 3 appointments that were in Reject tier (greg85md, galenwms, sbegum27)

### 7. eHawk Upstream + Form Signal Softening (ALL VERTICALS)
- **Context**: eHawk fraud detection runs upstream before leads reach the LLM
- **Change**: Added note to Form Behavior section that upstream fraud detection handles bots/fraud. LLM should focus on data quality signals, not fraud inference from form behavior.
- **Specific**: typing_paste changed from moderate concern to "slight concern — note but don't hard-penalize". autofill_only changed to "normal" (data shows 21.7% appt rate, above average).
- **Kept**: pre-populated_only stays as INSTANT REJECT, bot_detected stays as INSTANT REJECT

### 8. Renter → Neutral in Solar (SOLAR ONLY)
- **Signal**: batchdata.owner_occupied = "confirmed_renter"
- **Data**: Renters 16.4% appt rate (n=73) vs owners 14.2% (n=429) — renters OUTPERFORM. Renters without equity override convert at 26.7% (n=15).
- **Change**: Changed from "bad (but check equity override)" to "NEUTRAL — do not penalize renters." Removed RENTER OVERRIDE section (no longer needed since renter is neutral). Removed "renter with no equity override" from Bronze tier.
- **Roofing**: KEPT as "STRONG NEGATIVE" — roofing data may differ (renters genuinely rarely convert for roof replacement)
- **Impact**: Leads like moisesjreyes, senovia520 (Bronze 35, confirmed_renter with good signals) would score higher

## Key Architectural Notes (ALL VERTICALS)

### What the LLM Can and Cannot Catch
1. **Catchable signals** (LLM should filter): Phone grade F, NonFixedVOIP, low activity, invalid phone/email, bot detected, pre-populated forms, solar/roof permit
2. **Partially catchable**: Disconnected phones (Grade F + low activity catches ~50%), wrong-person leads (name match false catches some)
3. **Uncatchable with current API stack**: Credit score inadequate (16 leads, all looked perfect), income level, some "not homeowner" cases where BatchData returns null

### BatchData Income/Demographics
- Full-attributes API key does NOT return demographics section (bd_income, bd_net_worth, etc.)
- 0% fill rate on new enrichment
- Legacy enrichment had 14.3% fill rate
- FullContact (deprecated) had income for 46/1227 leads
- **Decision**: Not worth adding to LLM prompt until demographics data source is available
- **If BatchData demographics add-on is enabled**: Map bd_income to LLM as household_income, keep same prompt rules ($25k instant reject, $35k concern, null neutral)
